About this page: Here, we have three optional elements: Music, Star and Flag. That means that child pages can either have these elements or not. Optional elements can be deleted from child pages and restored if needed. Optional elements are defined by Define optional area action in ACT / Editable areas. Let's take a look:

Try:

  1. Open services.html and delete one of the optional elements. Update components. See, the deleted element doesn't come back on update.
  2. Open contact.html and update components. Hm... Optional elements didn't appear. That's correct - they are optional. Because they didn't exist on the page, update didn't add them to the page. This will also happen if you add a new optional element to the master page. Updating child pages will not add this element to child pages.
  3. So how can we get optional areas back on the child page? One way: on child page use Page -> Restore optional areas. This will restore all optional areas on the page. Try this on contact.html. But what if you want to restore just one element?
  4. To restore a particular optional area only, go to optional area definition on the master page and check "Restore on update". Then open all child pages where you want this area to be restored and do Components -> Quick update. Then uncheck "Restore on update", otherwise the area will come back on any updated child page.
